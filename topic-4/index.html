Categorical Response

Many learning tasks involve the following: given the value of predictors XXX, make a good prediction of the outcome yyy which we denote by G^\hat{G}G^.

In some cases, the outcome yyy is categorical, which is typically represented numerically by codes. For example, a categorical outcome with binary response is often represented by 000, 111.

One approach is to treat the binary coded outcome yyy as a quantitative variable to get the prediction. Then, assign a class label to G^\hat{G}G^ according to whether the prediction is larger than δ\deltaδ, which is a constant in (0,1)(0, 1)(0,1) and is chosen case by case.

K-Nearest Neighbors Classification

There are nnn training points with features xxx and categorical response yyy. Each (xi,yi)(x_i, y_i)(xi​,yi​) is the information of an observation.

With the information xxx, the prediction on the category for the response is denoted as G^(x)\hat{G}(x)G^(x).

For simplicity, we only consider the cases of binary response (0,1)(0, 1)(0,1) only. Hence, the prediction, G^(x)\hat{G}(x)G^(x), is either 000 or 111.

The kkk-nearest neighbors method uses training points closest in feature space to xxx to find:

1k∑_i∈N_k(x)yi,\frac{1}{k} \sum\_{i \in \mathcal{N}\_k(x)} y_i,k1​∑_i∈N_k(x)yi​,

where N_k(x)\mathcal{N}\_k(x)N_k(x) is the neighborhood of xxx defined as the set of kkk closest points (in terms of Euclidean distance).

δ\deltaδ is a pre-chosen threshold, and the average above is converted to G^(x)\hat{G}(x)G^(x) according to the rule

G^(x)={1,if it’s >δ,0,otherwise.\hat{G}(x) = \begin{cases} 1, & \text{if it's } > \delta, \\ 0, & \text{otherwise}. \end{cases}G^(x)={1,0,​if it’s >δ,otherwise.​

For the K-NN diagrams, the purple dash line refers to Bayes Decision, which we regard as the gold standard.

When k=1k = 1k=1, the decision boundary is overly flexible and influenced by local features of a handful of training data points →\rightarrow→ low bias, high variance.

When k=100k = 100k=100, the method yields more stable but less flexible decision boundaries →\rightarrow→ high bias, low variance.

Note that large or small of kkk is with respect to nnn:
k=50k = 50k=50 is large for n=100n = 100n=100, but small for n=100000n = 100000n=100000.

Application: Stock Market Data

We will use the knn() function from the 'class' package in R to perform kkk-nearest neighbors classification and prediction. We need the following four inputs to knn():

  1. A matrix containing the predictors or features xxx associated with the training data
  2. A matrix containing the predictors or features xxx associated with the data for which we wish to make predictions
  3. A vector containing the class labels for the training data
  4. A value for kkk, the number of nearest neighbors to be used by the classifier

The first 3 items are all retrieved from the dataset.

Diagnostics (Understanding the Model)

When a classifier is built, it helps to assign class labels to person, item, or transaction. There is a need to evaluate its performance.

In general, for two class labels, CCC and C′C'C′, where C′C'C′ denotes "not CCC", some working definitions and formulas follow:

  • True Positive: Predict CCC, when actually CCC
  • True Negative: Predict C′C'C′, when actually C′C'C′
  • False Positive: Predict CCC, when actually C′C'C′
  • False Negative: Predict C′C'C′, when actually CCC

Confusion Matrix


TP and TN are the correct predictions. A good classifier should have large TP and TN; and has small numbers (ideally zero) for FP and FN.

Metrics for Diagnostics of Classifiers

There are many criteria to evaluate. For now, we consider 5 basic measurements below:

  1. Accuracy
  2. True Positive Rate (TPR)
  3. False Positive Rate (FPR - Type I error)
  4. False Negative Rate (FNR - Type II error)
  5. Precision

A well-performed classifier should have a high TPR (ideally 1) and a low FPR and FNR (ideally 0).

Accuracy

The accuracy (or the overall success rate) is a metric defining the rate at which a model has classified the records correctly.

It is defined as the sum of TP and TN divided by the total number of instances:

Accuracy=TP+TNTP+TN+FP+FN×100%\text{Accuracy} = \frac{TP + TN}{TP + TN + FP + FN} \times 100\%Accuracy=TP+TN+FP+FNTP+TN​×100%

A good model should have a high accuracy score, but having a high accuracy score alone does not guarantee the model is well established.

To find the accuracy of our model above:

sum(diag(confusion.matrix))/sum(confusion.matrix) # 0.515873

Accuracy=(55+75)/252≈51.59%\text{Accuracy} = (55+75)/252 \approx 51.59 \%Accuracy=(55+75)/252≈51.59%
Which means that 51.59% of the observations are correctly predicted.

True Positive Rate

The true positive rate (TPR) shows the proportion of positive instances the classifier correctly identified:
TPR=TPTP+FN\text{TPR} = \frac{TP}{TP + FN}TPR=TP+FNTP​

False Positive Rate

The true positive rate (TPR) shows the proportion of positive instances the classifier correctly identified, and is also called the false alarm rate or Type I error rate:
FPR=FPFP+TN\text{FPR} = \frac{FP}{FP + TN}FPR=FP+TNFP​

False Negative Rate

The false negative rate (FNR) shows the percent of positives the classifier marked as negatives and is also known as the miss rate or Type II error rate.
FNR=FNFN+TP\text{FNR} = \frac{FN}{FN + TP}FNR=FN+TPFN​

Precision

Precision is the percentage of instances that are actually positive among the marked positives:
Precision=TPTP+FP\text{Precision} = \frac{TP}{TP + FP}Precision=TP+FPTP​

N-fold Cross Validation

  1. The entire data set is randomly split into NNN data sets of approximately equal size.
  2. (N−1)(N-1)(N−1) of these data sets are treated as the training data set, while the remaining one is the test data set. A measure of the model error is obtained.
  3. This process is repeated across the various combinations of NNN data sets taken (N−1)(N-1)(N−1) at a time.
  4. The observed NNN models' errors are averaged across the NNN folds.

5-fold Cross-Validation for KNN with k=1,5,10k=1, 5, 10k=1,5,10

  1. Extract the features and response:
X = market[,c("Lag1","Lag2","Lag3","Lag4","Lag5")] # columns of features
Y = market[,c("Direction")] # response
  1. Create sample:
n_folds=5

# dim(market)[1] gives number of rows
# gives a vector containing equal number of 1s to 5s, shuffled
folds_j <- sample(rep(1:n_folds, length.out = dim(market)[1]))
table(folds_j)

folds_j
  1   2   3   4   5
250 250 250 250 250
  1. Create vector to store error rate and accuracy of each fold:
err=numeric(n_folds)
acc=numeric(n_folds)
  1. Loop through each fold to populate error rate and accuracy:
for (j in 1:n_folds) {
    test_j <- which(folds_j == j) # vector of indices corresponding to the fold

    train.x = X[-test_j, ] # item 1
    test.x = X[test_j, ] # item 2
    train.y = Y[-test_j ] # item 3

    # KNN with k = 1, 5, 10, etc
    pred <- knn(train.x, test.x, train.y, k=1)

    # Y[test_j] == pred returns a logical vector (TRUE == 1, FALSE == 0)
    err[j]=mean(Y[test_j] != pred)
    acc[j]=mean(Y[test_j] == pred)
}

error=mean(err); error # this is the avarage error rate over 5 folds

accur=mean(acc); accur # this is the avarage accuracy over 5 folds
